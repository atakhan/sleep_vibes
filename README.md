# Sleep Vibes

Веб-приложение с процедурной генерацией CSS анимации и звуков на стороне клиента.

## Структура проекта

- `index.html` - главный HTML файл со всеми экранами
- `css/style.css` - стили приложения
- `js/app.js` - основная логика приложения и навигация
- `js/screens.js` - управление экранами
- `js/generator.js` - процедурная генерация анимаций и звуков
- `docker-compose.yml` - конфигурация Docker Compose
- `nginx.conf` - конфигурация Nginx

## Экраны приложения

1. **StartSleepScreen** - экран начала сессии с настройками
2. **SessionScreen** - экран с процедурной анимацией и звуком
3. **StillAwakeScreen** - экран для подтверждения, что пользователь все еще не спит (с таймером на 15 секунд)

## Запуск через Docker Compose

```bash
docker-compose up -d
```

Приложение будет доступно по адресу: http://localhost:8080

## Остановка

```bash
docker-compose down
```

## Заглушки для серверных данных

В файле `js/app.js` помечены места (TODO комментарии), где должны быть запросы к серверу:
- При начале сессии (`startSleep()`)
- При нажатии "все еще не сплю" (`handleStillAwake()`)
- Загрузка списка звуков и анимаций (в методах `updateSoundDisplay()` и `updateAnimationDisplay()`)
